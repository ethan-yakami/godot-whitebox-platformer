# 白盒原型资产需求文档

## 第一版资产原则

第一版只要求程序白盒可玩。所有资产先用简单色块、矩形、文字和调试 UI 表达功能。正式美术可在玩法闭环确认后替换。

## 资产清单

| 资产 | 白盒表现 | 最终美术需求 | 程序依赖 |
| --- | --- | --- | --- |
| 玩家 | 蓝色矩形，48x64 | 西幻冒险角色，待机/跑/跳/冲刺/受击 | `CharacterBody2D` 碰撞体 |
| 普通平台 | 灰色矩形 | 草地、岩石、木板等关卡主题平台 | 静态碰撞 |
| 长/短平台 | 灰色不同长度矩形 | 需匹配 32px 网格 | 静态碰撞 |
| 尖刺 | 红色细矩形 | 尖刺动画或静态图 | `Area2D` 扣 1 血 |
| 移动尖刺 | 红色障碍，后续可动画 | 移动机关 | 触发范围、移动速度、CD |
| 水流 | 半透明蓝色区域 | 湖泊/水流方向视觉 | 持续水平推力 |
| 特殊障碍 | 深红区域 | 岩浆/污染地形 | 扣 2 血 |
| 移动平台 | 橙色矩形 | 可移动机关平台 | 往返移动 |
| 塌陷平台 | 粉色矩形 | 踩踏破碎动画 | 首版可暂缓 |
| 弹跳平台 | 绿色矩形 | 弹簧/蘑菇等 | 给玩家向上速度 |
| 路灯 | 黄色竖条 | 存档路灯和光圈 | 自动存档、回血 |
| 浆果 | 红色小方块 | 可采集浆果 | 回复 1 血 |
| 石碑 | 蓝灰色块 | 文本交互物 | 显示文字，首版可调试输出 |
| 献祭坛 | 蓝色块 | 传承祭坛 | 第五关授予传承 |
| 普通怪 | 红色矩形 | 第四关怪物动画 | 巡逻、触发、攻击 |
| Boss | 大号红色矩形 | 第五关前辈 Boss、起点污染 Boss | 血量、攻击、胜利触发 |
| 残影 | 半透明蓝色敌人 | 玩家影子/灵体效果 | 最多 2 个，死亡点生成 |
| HUD | 文本标签 | 血量、限制按钮、暂停菜单正式 UI | 实时显示状态 |

## 美术交付建议

- 优先交付玩家、平台、尖刺、路灯、限制 UI。
- 所有 2D 资产建议围绕 32px 网格制作。
- 玩家碰撞体暂定 3x4 格参考值过大，白盒先用 48x64，正式美术后再校准。
- 特效第一版只需要传承获得、受伤、存档激活三个反馈。
