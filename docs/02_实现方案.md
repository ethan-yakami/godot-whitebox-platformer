# 白盒原型实现方案

## 技术选择

- Godot 4.4/4.5。
- GDScript。
- 玩家使用 `CharacterBody2D + move_and_slide()`。
- 伤害、交互、水流、出口使用 `Area2D`。
- 关卡白盒由 `scripts/main.gd` 根据 `scripts/level_data.gd` 动态生成。

## 系统划分

| 系统 | 文件 | 职责 |
| --- | --- | --- |
| 全局状态 | `scripts/game_state.gd` | 阶段、关卡、血量、传承、能力、存档点、残影列表 |
| 输入限制 | `scripts/input_limiter.gd` | 配置每关限制，检查和消耗按键次数 |
| 玩家控制 | `scripts/player.gd` | 移动、跳跃、二段跳、冲刺、受伤、复活 |
| 关卡配置 | `scripts/level_data.gd` | 平台、机关、敌人、出口、去返程限制 |
| 主流程 | `scripts/main.gd` | 加载关卡、生成白盒、奖励、去返程切换 |
| HUD | `scripts/hud.gd` | 血量、关卡、阶段、能力、限制次数、调试信息 |
| 机关 | `scripts/hazard.gd` | 伤害区、水流、弹跳等 Area 行为 |
| 交互 | `scripts/interactable.gd` | 浆果、路灯、出口、献祭坛 |
| 敌人/残影 | `scripts/enemy.gd`, `scripts/ghost.gd` | 巡逻、追击、碰撞伤害 |
| 移动平台 | `scripts/moving_platform.gd` | 白盒往返移动平台 |

## 数据流

1. `Main.load_level()` 从 `LevelData` 读取当前关卡。
2. `InputLimiter.configure()` 读取当前阶段限制。
3. `Player` 在输入时调用 `consume_action()`，次数不足则阻止动作。
4. `Hazard`、`Enemy` 调用 `Player.take_damage()`。
5. `GameState.damage()` 处理血量；死亡时记录残影并发出信号。
6. `Main` 收到死亡信号后生成残影并让玩家回到 `checkpoint_position`。
7. `Hud` 每帧显示当前调试状态。

## 白盒实现约束

- 所有白盒块使用 `ColorRect + CollisionShape2D`。
- 关卡尺寸和布局先参考 PPT，不追求像素级复刻。
- Excel 数值作为默认值，手感不佳时允许程序调整，但要回填配置表。
- Boss 第一版只做大号敌人和传承触发，不做复杂招式。
- 读档菜单第一版只保留暂停面板说明，实际死亡自动复活。

## 手感参数

玩家参数放在 `Player` 导出变量：

- `speed = 256`
- `jump_velocity = -560`
- `dash_speed = 720`
- `dash_duration = 0.14`
- `dash_cooldown = 0.5`
- `coyote_time = 0.1`
- `jump_buffer_time = 0.12`

这些参数用于快速迭代平台跳跃手感，参考 Celeste 等平台游戏常用的宽容输入设计。
