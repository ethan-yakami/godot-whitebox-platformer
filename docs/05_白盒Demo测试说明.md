# 白盒 Demo 测试说明

## 启动

1. 用 Godot 4.4/4.5 打开 `project.godot`。
2. 运行主场景 `scenes/main.tscn`。
3. 若脚本报错，优先把 Godot 输出面板错误复制给程序。

## 操作

| 按键 | 行为 |
| --- | --- |
| A/D | 左右移动；受关卡限制时按下一次消耗一次 |
| H | 跳跃；二段跳消耗 2 次跳跃限制 |
| Space | 冲刺；获得传承前不可用 |
| J | 攻击/交互 |
| Esc | 暂停/继续 |
| L | 读取最近路灯；不重置限制次数 |
| R | 重开当前关；重置当前关限制次数 |
| Shift+R | 重开整轮白盒 Demo |

## 推荐测试路径

1. 第一关对照 PPT4：起始视野框、尖刺坑、浆果、粉色崩塌平台、高台终点都应出现。
2. 通过第一关后确认 HUD 显示二段跳为 `true`。
3. 第二关对照 PPT5-6：两页湖泊被合并为长关，底部/中部浅黄水流会持续向右推。
4. 第二关踩粉色塌陷平台，确认它先能站立，约 0.35 秒后消失。
5. 第三关对照 PPT7：确认三层结构、底层水流、中层浆果/崩塌平台、上层终点。
6. 第三关确认 A 键限制以“按下一次”消耗，不是按住每帧消耗。
7. 第四关对照 PPT8-9：先到石碑并按 J 解锁攻击，再进入黑球敌人战斗区。
8. 第五关对照 PPT10：低天花板、Boss 平台、右侧献祭石和台阶尖刺都应出现。
9. 第五关先不打 Boss 直接碰祭坛，确认提示“需要先击败 Boss”。
10. 第五关击败 Boss 后确认限制解除，再与蓝色祭坛交互获得传承。
11. 任意关按 L，确认读档回最近路灯、血量回满、限制次数不重置。
12. 任意关按 R，确认重开当前关、限制次数恢复。
13. 任意关按 Shift+R，确认回到第一关去程，能力和残影清空。
14. 获得传承后确认 HP 变为 5、冲刺可用、阶段变为 return。
15. 返程故意死亡，确认回最近路灯，并生成最多 2 个蓝色残影。
16. 返回第一关出口位置，确认显示胜利文本。

## 白盒颜色说明

- 蓝色矩形：玩家。
- 灰色矩形：平台。
- 红色区域：伤害。
- 浅蓝区域：水流。
- 绿色短条：弹跳垫。
- 粉色短条：塌陷平台。
- 黄色竖条：路灯存档点。
- 红色小块：浆果或敌人。
- 大号红色敌人：Boss。
- 半透明蓝色敌人：残影。

## 已知限制

- 关卡图是白盒近似，不是 PPT 的精确复刻。
- Boss 只有占位巡逻/碰撞伤害，没有正式技能。
- 石碑文字与正式设置菜单暂未做。
- 起点 NPC 污染 Boss 仍是后续内容。
- 当前没有 Godot 命令行自动化测试，需用编辑器人工跑一遍上述路径。
